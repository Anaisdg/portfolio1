SET Card Game Image Recognition

An image recognition web application that allows a user to input a picture of the board and finds all the SETs according to the rules of the game.

The game SET is a popular card game that was invented by Population Geneticist, Marsha Jean Falco, who was trying to understand if German Shepherds who get epilepsy inherit it. The game evolved out of a coding system that she used in her job to answer her question about inheritance.
The object of the card game SET is to identify a 'set' of three cards from 12 cards laid out on the table. Each card has a variation of the following four features:
(A) Color: Each card is red, green, or purple.
(B) Symbol: Each card contains ovals, squiggles, or diamonds.
(C) Number: Each card has one, two, or three symbols.
(D) Shading: Each card is filled, empty, or striped.

A SET consists of three cards in which each feature is EITHER the same on each card OR is different on each card. That is to say, any feature in the 'Set' of three cards is either common to all three cards or is different on each card.

If you've ever played the game, you know its not easy. Here's an example of picture of the board that a user might input. In this exammple, there are 5 sets total. Can you find them all? Can't find any? You're not alone. This game is played in with group, but often times all the players can't find a SINGLE SET .


Turns out, that's because humans aren't really that good at finding SETs. There are 81 unique cards in this game, which yields 1080 unique SETs. The odds against there being no SET in 12 cards when playing a game of SET start off at 30:1 for the first round. Then they quickly fall, and after about the 4th round they are 14:1 and for the next 20 rounds they slowly fall towards 13:1. So for most of the rounds played, the odds are between 14:1 and 13:1. The odds against there being no SET in 15 cards when playing a game are 88:1.
To help you and your friends find the SET that probably exists and prevent you from pulling your hair out, my team and I made a web appication that allows you to upload a picture of your board and finds the SETs for you!



How it works:

Image Uplaod: The user takes an uplaods a piture of their board.
Card Identification & Image Segmentation: With OpenCV, the contours of the edge of the cards are detected, the cards are cropped (image segmentation), and temporarily stored in a directory.
Card Prediction and Prediction Decoding: The color of the shapes in the cropped cards are identified using OpenCV. The cropped card is then run through a Keras model which predicts which class the card belongs to. The model we made has 92% accuracy for categorical classification. After the prediction is made, a decoder asigns the appropriate label which contains information on the rest of the card features (shape, number of symbols, and shading)
Game Logic: Once all of the card labels have been collected, they are sent through the game logic which outputs a list of SETs.
API Call & Display: This list is used to retrieve pictues of the cards that are in the players SETs from a custom API. Finally, the SETs are displayed to the user, and the user finds peace.

This game, although seemingly simple, came with its fair load of challenges. Unfortunately, our Keras model, even at 92% accuracy, isn't working well enough for us. We won't deploy the app until we can get 98% accuracy -- the accuracy we got for a proof of concept 12 card classifier.
